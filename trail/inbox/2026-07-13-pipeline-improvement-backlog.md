## 확정 우선순위 (codex 권장 1→2→3→5→4)

1. **리뷰 증거 기계화 + 리뷰 직전 준비도 사전검사 (한 묶음)**: `scripts/rein-codex-review.sh` 가 diff 기준점 기반 변경 파일 목록·증거 manifest(command/exit_code/output)를 직접 생성, 훅이 누락 검사. 프롬프트 규약만으로는 불충분(git status 는 범위 정본 부적합 — 커밋된 변경 누락/무관 변경 혼입). 정량·PASS 주장에 한정. 기대: 보고 오류발 반려 제거(실측 반려 20→16, 웨이브당 5~15분).
2. **혼합형 통합 리뷰 체크포인트**: 고정 아키텍처 경계 checkpoint 기본 + plan 태스크 `cross_cutting`(직렬화·원자성·계상·스키마) 태그 시 조건부 추가 발동. 순수 조건부는 기각 — 구현자 사후 판정은 "미지의 결합" 을 못 잡음(실측 6건이 반례). 기대: 실결함의 37.5% 조기 탐지, 수정 파급 축소.
3. **워커 ACK·lease 재전송**: parallel-execute 에 task ID 수신확인 + 점유 기한 + 완료 manifest + 멱등 재전송. 파일 존재 확인만으로는 부족(이전 산출물 오인·중복 워커 위험). 기대: 지시 유실 12회→0, 30분+ 회수.
4. **배선 의무 + 계약 테스트**: plan validator 가 신규 컴포넌트의 진입점·소비자·계약 테스트 선언 요구, barrier 에서 실행. grep 참조 검사는 보조 lint 로만(동적 등록/DI 누락, 참조만 있고 미실행 통과).
5. **보안 도장 복합 해시**: 소스+의존성+보안 설정+규칙 버전 composite digest, 유효성/신선도 분리. v1.5.8 잔존 항목과 합류. 우선순위 최하(실측 오탐 비용 ~10분).
