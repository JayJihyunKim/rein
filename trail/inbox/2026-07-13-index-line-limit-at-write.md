# index 줄 수 한도 작성 시점 강제 — 완료 기록 (2026-07-13)

## 요약

사용자 요청("작성 후 25줄 검증 왕복 대신 애초에 맞추게"). 신규 훅 `plugins/rein-core/hooks/pre-edit-index-lines.sh` 가 trail/index.md 의 Write/Edit/MultiEdit **결과 줄 수**를 시뮬레이션해 5~25줄 위반이면 편집 자체를 거부 + 압축 안내. 종료 게이트 검사는 이중 방어로 유지. codex 리뷰 5라운드 PASS + 보안 리뷰 PASS(발견 0), 테스트 15픽스처 + 전 스위트 GREEN.

## 리뷰가 잡은 실결함 (왕복의 가치 실증)

- R1 Medium: 상대경로 입력을 훅 CWD 기준으로 해석해 대상인데도 통과 → PROJECT_DIR 기준 join 수정 + 회귀 픽스처.
- R2 High: MultiEdit 실계약(edits[*].file_path, 여러 파일 혼합)을 미지원 — 실제 MultiEdit 는 통과했을 것 → per-edit 경로 + top-level fallback + 혼합 파일 제외 픽스처.
- R3→R4 합의: 문자열 fast-path 의 symlink 별칭 누락은 위협모델(정직한 에이전트 규율)·hot-path 성능 근거로 **수용 한계** 로 기록(훅 헤더 + DoD 비고). 종료 게이트가 실파일 재검사로 적발.
- 줄 수 산식을 종료 게이트의 wc -l(개행 수)과 동일하게 정렬(자체 발견) — 산식 불일치로 인한 잔존 왕복 차단.
