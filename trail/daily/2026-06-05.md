## session-state-git-snapshot

### DoD

## 목표

세션 상태 파일 `trail/index.md` 의 git/릴리스 서술이 손글씨라 실제 git 과 어긋나는 신선도 오류 클래스를 제거한다. 훅이 git 객관 사실(branch / HEAD short sha / dirty·clean / ahead·behind / 최신 태그 / UTC 타임스탬프)을 `.rein/state/git-snapshot.md`(gitignored)에 매 세션 자동 생성하고, SessionStart 가 index 블록 직후 주입한다. index 는 서술 전용으로 남기고, 충돌 시 자동 스냅샷(궁극적으로 살아있는 git)이 권위본임을 규칙으로 명문화하며, 손글씨 휘발 git 수치가 잔존하면 Stop 이 비차단 advisory 로 경고한다.

## 완료 기준 (acceptance)

1. 신규 공유 lib `plugins/rein-core/hooks/lib/git-snapshot.sh` — `rein_write_git_snapshot <project_dir>` 가 6필드를 `.rein/state/git-snapshot.md` 에 tmp→mv 원자 재생성. fresh-write-or-clear(실패/non-git 시 stale 파일 rm -f). detached HEAD 라벨 + ahead/behind 생략. 최신 태그 "로컬 최근 생성, 최고 semver 아님" 라벨. 네트워크 명령 0건. 항상 비차단(return 0). (GS-1)
2. SessionStart·Stop 양쪽이 lib source 후 호출. Stop 은 no-source-edit exit(stop-session-gate.sh:158) **이전**에 bootstrap-complete 가드 하 호출(push/릴리스만 한 세션도 갱신). SessionStart 는 emit_file_block "trail/index.md"(line 346) 직후. (GS-2)
3. SessionStart 가 index 블록 직후 git-snapshot 을 별도 블록으로 주입 — **파일 존재 시에만**(fresh-write-or-clear 로 stale 주입 불가). early-exit 경로 미주입. (GS-3)
4. Stop 좁은 비차단 advisory — index 서술의 손글씨 휘발 git 수치(앵커 정규식 목록, bare dirty/clean 금지) 잔존 시 stderr 1줄. 의미 서술 미발화(음성). exit code / block JSON 불변. post-158 경로, 신선도검사 이후. (GS-4)
5. `.gitignore` 에 `/.rein/state/` 추가 + `.claude/rules/branch-strategy.md` 에 `.rein/state/` main 제외 + mirror strip 명시. (GS-5)
6. 권위 규칙 1줄을 `operating-sequence.md` step 11 + `trail/index.md` 헤더 노트 양쪽 배치. (GS-6)
7. 회귀 테스트 신규 `tests/hooks/test-git-snapshot.sh` — assert (a)~(h) + codex Low(index 없고 trail 만 있는 경우) + Stop High(no-source-edit mtime 갱신). `tests/hooks/run-all.sh` 등재. 전체 스위트 green. (GS-7)
8. 버전 등급 단일 권고 = minor 기록(main 머지 직전 메인테이너 확정). (GS-8)
9. 통합 코드 리뷰(codex) + 보안 리뷰 1회 통과 후 커밋.

## 무엇을 / 왜

세션 종료 시 index.md 의 git/릴리스 서술이 손글씨라 실제 git 과 어긋나는 신선도 오류(실측: "미push 5건" vs 실제 ahead 6)를 클래스 단위로 제거. git 객관 사실을 훅이 직접 도출해 자동 기록하고, 손으로 쓰는 건 의미 서술만 남김.

## 구현 (GS-1~GS-8)

- **GS-1** 신규 lib `plugins/rein-core/hooks/lib/git-snapshot.sh` — `rein_write_git_snapshot` 가 6필드(branch[detached 라벨]/HEAD short/dirty·clean+수/ahead·behind[로컬 추적, 네트워크 없음]/최신 태그[creatordate, describe 아님]/UTC)를 `.rein/state/git-snapshot.md` 에 tmp→mv 원자 기록. fresh-write-or-clear(non-git/unborn HEAD/실패 시 stale rm -f). BC-INFO1 env 위생(`_rein_git_clean`). 항상 return 0.
- **GS-2/3** SessionStart(`session-start-load-trail.sh`): lib source(fail-open) + index 블록 직후 재생성·주입(파일 존재 시만, `~~~` 펜스).
- **GS-2/4** Stop(`stop-session-gate.sh`): no-source-edit exit(line 158) **이전** bootstrap-complete 가드 하 재생성(push/릴리스만 한 세션도 갱신) + 신선도검사 이후 좁은 비차단 advisory(앵커 정규식, bare dirty/clean 미매칭).
- **GS-5** `.gitignore` `/.rein/state/` + `branch-strategy.md` 제외 명시.
- **GS-6** 권위 규칙 1줄: `operating-sequence.md` step 11 + `index.md` 헤더.
- **GS-7** 회귀 `tests/hooks/test-git-snapshot.sh` (a~i + g2 + index-less + no-src-edit) 17 assert + run-all 등재.
- **GS-8** 버전 등급 권고 = minor (main 머지 직전 확정).
