## 배경

emitter demo 검토 중 사용자 지적: 케이스 2 (fail-closed)에서 사용자 화면에 뜨는
메시지가 `rein-json-deny-emitter: empty reason — refusing to emit (fail-closed,
exit 2)` 같은 기술 문구라 불친절. `exit 2` stderr는 Claude 미경유 (공식 문서
"not to Claude")라 다국어 변환이 안 됨 — 영문이되 친절해야 한다는 요구.

## 변경 내용

6개 fail-closed 메시지를 `[rein] A tool call was blocked, ... It stays blocked
for safety.` 패턴으로 재작성:
- echo 3개 (shell): empty reason / no usable python3 / empty JSON output
- python `sys.stderr.write` 3개: reason empty after sanitisation / JSON
  serialisation failed / produced JSON re-parse failed

각 메시지는 "무엇이 차단됐고 / 왜 / 안전을 위해 차단 유지됨"을 자연 영문 문장으로
설명. python3 부재 케이스는 "Installing python3 will restore the detailed
message" 같은 사용자 조치도 안내.

## 미반영 (2단계 brainstorm 이월)

데모 중 추가로 드러난 reason 구조 결함 (신뢰된 사유+행동 vs DATA로 격리할 입력의
2슬롯 분리) — `daily/2026-05-16.md`의 emitter 항목 "2단계 설계 발견" 참조.
