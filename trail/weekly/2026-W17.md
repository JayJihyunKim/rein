1. Bash 도구가 `timeout 420000ms` 같은 장기 실행 명령을 **자동으로 `run_in_background: true`** 로 전환 (harness 정책)

**반복 패턴**: 사용자 관찰 — 이전 세션들에서도 동일 증상 재발.

**결정**: `codex` / `codex-review` / `codex-ask` 계열은 Bash 도구 background auto-switch **예외**. foreground + TTY + 직접 파일 출력 강제. 다른 장기 명령은 기존 `rein job` / Bash background 규칙 유지.
