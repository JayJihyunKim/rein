## 배경

`rein-bootstrap-project.py:135-141` 는 부트스트랩 시점에 "project-dir must be the git root" 를 강제. 사용자는 git root 에 trail/ 을 만들 수밖에 없음.

그런데 런타임 gate `bootstrap-check.sh` 는 정반대 contract:

```bash
# bootstrap-check.sh:118-131
if [ -n "$stdin_cwd" ]; then
  resolved="$stdin_cwd"    # ← 1순위 (subdir 가 잡힘)
  source="stdin"
else
  git_root="$(git rev-parse --show-toplevel ...)"
  ...                       # ← 2순위로 밀려 있음
```

Claude Code 가 PreToolUse:Bash hook envelope 의 `cwd` 로 **셸 CWD** 를 넘기는데, Bash 도구는 셸 CWD 가 호출 간 유지됨. 사용자가 monorepo subdir (`apps/web`) 에서 `cd` 한 번 하면 그 후 모든 Bash 호출의 envelope cwd = subdir. bootstrap-check 가 그걸 1순위로 채택 → `apps/web/trail/` 부재 → exit 10 → Bash 차단.

정상적으로 git root 에 bootstrap 한 사용자도 monorepo subdir 에서 Bash 한 번 돌리면 hook 이 막음. 사용자가 보고한 실제 증상 (contextOS monorepo / `apps/web` Phase C 검증 중 차단).
