# DoD: Stop hook session_end "퇴근 도장" 의미 redefine

## 배경

직전 세션이 `git commit` block 으로 끝나면서 `session_end=true` 를 못 찍어, 새 세션 SessionStart 가 "비정상 종료 감지" false alarm 출력. codex-ask 두 번 (gpt-5.4/high → gpt-5.3-codex/high) 진단 결과:

- session_end 기록은 stop hook 의 정상 종료 경로 (line 332-343) 에만 있어 6개 exit 경로 중 5개가 도장 못 찍음
- `rein-aggregate-incidents.py:270` 가 매 aggregate 호출마다 session_end=False hardcoded 로 덮어써서, stop hook 이 도장 찍어도 다음 SessionStart recovery aggregate 가 지움
- trap / SessionStart reset / aggregate 의 다중 writer 가 lock 없이 snapshot 을 건드리면 race condition 가능

## 목적

`session_end` 의 의미 redefine:
- **기존**: "Stop hook 이 정상 11단계 모두 통과해서 끝까지 갔음"
- **변경**: "Stop hook 이 어떤 경로로든 실행은 됨"

→ false 로 남는 것은 진짜 hook 미호출 (Ctrl+C / 터미널 닫기 / Claude Code crash / SIGKILL) 만. SessionStart 의 "비정상 종료 감지" 메시지가 정확해진다.

## 변경 범위

1. **`scripts/rein-aggregate-incidents.py`** — 신규 subcommand `--set-session-end true|false`. flock 안에서 atomic write (단일 writer 경로 통합). 기존 aggregate 의 snapshot 작성 시 prev session_end 보존
2. **`.claude/hooks/stop-session-gate.sh`** — `trap EXIT` 함수 추가. 어떤 경로로 끝나든 (BYPASS env / no-src-edit / pending-block / loop-3x / inbox-or-index missing / 정상) `--set-session-end true` 호출. 기존 line 329-343 의 정상 종료 시 session_end 기록 블록은 trap 으로 통합되며 제거
3. **`.claude/hooks/session-start-load-trail.sh`** — ABNORMAL 체크 직후 + recovery aggregate 호출 **전** 에 `--set-session-end false` 호출 (이번 세션 도장 초기화)
4. **plugins/rein-core 미러** — `plugins/rein-core/hooks/stop-session-gate.sh` + `plugins/rein-core/hooks/session-start-load-trail.sh` byte-identical 미러
5. **테스트**:
