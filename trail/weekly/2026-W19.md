## 배경

v2.0 plugin-first 재구조화 (`docs/specs/2026-04-27-plugin-first-restructure.md`) 이후
`trail/` 은 `plugins/rein-core/scripts/rein-bootstrap-project.py` 가 사용자 git root
에 직접 생성하도록 contract 가 박혀 있다. 해당 스크립트는 `is_plugin_storage()` 로
`.claude/plugins/**` 경로에서 실행되면 `fail()` 까지 한다 — "trail/ 은 plugin
저장소가 아니라 사용자 git root 에 속한다" 는 명시적 design intent.

그런데 main 에는 v1 (scaffold) 시대 잔재로 다음 7개 파일이 tracked 상태:
- `trail/index.md` (빈 템플릿, 2 lines)
- `trail/{agent-candidates,daily,dod,inbox,incidents,weekly}/.gitkeep` (6개 — `decisions` 폴더는 원래 .gitkeep 없음)

이 잔재는 `/plugin marketplace add` 가 source git 레포를 통째 clone 할 때
`~/.claude/plugins/marketplaces/rein/trail/` 로 따라 들어와서 bootstrap 스크립트의
contract 와 정면으로 충돌한다. codex second opinion (gpt-5.5, medium effort,
2026-05-06) 도 사용자 지적 ("trail 은 사용자 작업 디렉토리에 생성되도록 한
것 아니냐") 이 맞다고 판단.
