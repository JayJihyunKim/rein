## 목표 (Why)

dev 에 안착한 보안 게이트 hardening 3건 — SR-1 (spec-review stale `.reviewed` 우회), GE-1 (selector path containment), GE-2 (stamp diff_base ancestor 검증) — 을 patch 릴리스 v1.3.5 로 사용자에게 도달시킨다. 세 변경 모두 user-facing hook 동작 변화 (plugin 으로 ship) 이므로 Rule A patch.

**Rule B 주의**: v1.3.4 가 오늘(2026-05-22) 릴리스됨. Rule B 는 같은 날 2차 main 머지를 핫픽스로 한정하나, SR-1/GE 는 코드리뷰 발견 잠재 갭(in-the-wild regression 아님)이라 핫픽스 예외엔 미해당. **사용자가 Rule B override 를 명시 승인** (2026-05-22 "지금 v1.3.5 로 릴리스"). 커밋/CHANGELOG 에 user-directed same-day release 로 정직 기록.

## 성공 기준 (Acceptance)

1. `plugins/rein-core/.claude-plugin/plugin.json` version = `1.3.5`.
2. `scripts/rein.sh` `VERSION="1.3.5"` — plugin.json 과 parity (publish VER-1 게이트 전제).
3. `CHANGELOG.md` v1.3.5 섹션 — user-facing 항목만 (SR-1/GE-1/GE-2), Rule C.
4. `README.md` / `README.ko.md` "Latest release" = v1.3.5 (KR/EN parity, 간략 + CHANGELOG 링크).
5. codex 리뷰 PASS + `.codex-reviewed` fresh stamp (버전 표면 edit 이 생성한 `.review-pending` 해소).
6. dev commit + push → main 선별 체크아웃(branch-strategy 포함 목록) → annotated tag `v1.3.5` → push (dev/main/tag).
7. main 선별 체크아웃 제외: `need-to-confirm.md`, `confirmed.md`, `rein-v1.4-improvement-plan.html` (루트 임시 노트), DoD/inbox 의 메인테이너 @import.
