## Round 3 finding 처리

**HIGH (lock-acquire 실패 시 envelope skip — 실제 결함, FIXED)**:
`read_effective_mode` 는 lock 획득 실패 시 stdout 에 "answer" 를 출력하고 non-zero
로 종료한다. 두 envelope hook 의 capture 가 `$(read_effective_mode || echo "")`
로 exit status 를 무시 → lock 실패에도 "answer" 를 신뢰해 envelope 를 skip
(fail-soft 위반). **Fix**: 세 state-aware hook 모두 `if state_is_valid &&
_mode=$(read_effective_mode 2>/dev/null) && [ "$_mode" = "<target>" ]` 패턴으로
교체 — read 가 exit 0 일 때만 mode 를 신뢰, 실패 시 legacy path.

**HIGH (committed diff 에 변경 없음 — pre-commit artifact)**:
위 "리뷰 대상" 절 참고. working tree 에 변경이 모두 존재하며, 커밋은 이 리뷰
PASS + security stamp 이후 단계다.
