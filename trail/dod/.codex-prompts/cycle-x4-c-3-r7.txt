## "answer" fallback 부류 — 종결 논증 (Round 1~6 누적)

read_effective_mode (fast-path 가 신뢰하는 유일 함수, 호출처 3 hook 모두
`state_is_valid && read 성공 &&` 게이트) 가 "answer" 를 잘못 내던 모든 경로:

| 경로 | 종결 |
|---|---|

| malformed JSON (R2) | state_is_valid: json.load + dict |
| unknown schema_version (R2) | state_is_valid: schema_version==1 |
| lock 실패 → "answer"+rc2 (R3) | hook capture 가 exit status 신뢰 |
| python 부재 → "answer"+rc0 | state_is_valid: `_state_machine_python` 실패→rc1 |
| schema-v1 + 필드 타입 except (R4) | state_is_valid: last_drain_seq int / dirty_files list |
| schema-v1 + 비-enum/whitespace mode word-split (R5) | state_is_valid: mode∈enum |
| schema-v1 + required 필드 누락 (R6) | state_is_valid: §2 required 전수 present |

§2-valid state 에서 read_effective_mode 는 (lock OK 시) 항상 단일-단어 enum mode 를
clean parse 로 rc0 반환 → fast-path 는 genuine mode 일 때만 skip. read_effective_mode
의 모든 fallback 진입점이 (a) state_is_valid 사전 차단 또는 (b) exit-status 신뢰로
무력화됨. 남은 누출 경로가 있다면 그것은 §2 검증으로 막을 수 없는 종류 (= 접근법
