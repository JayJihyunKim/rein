## Round 4 HIGH (class fix — 사용자 승인된 근본 수정)

Round 4 finding: schema_version==1 은 통과하지만 필드 값이 깨진 state
(`{"schema_version":1,"mode":"source_edit","dirty_files":[],"last_drain_seq":"bad"}`)
에서 read_effective_mode 의 `int(last_drain_seq)` 가 except → "answer" 출력 +
return 0 → 두 envelope hook 이 잘못 skip.

이는 Round 1~4 가 매번 발견한 **단일 부류** 의 마지막 진입점이었다: "answer" 가
(1) 진짜 모드값 + (2) 모든 read/parse 실패의 fallback 으로 이중 사용되어, fast-path
가 둘을 구별 못 함. 인스턴스가 아니라 **부류** 를 닫기 위해 `state_is_valid` 를
강화 (사용자 승인).

**Fix (state-machine.sh `state_is_valid`)** — schema_version==1 에 더해, fast-path
mode 도출이 의존하는 모든 필드의 타입을 검증:
- `mode` — present + non-empty str (누락/비문자열 → legacy. mode-less 는 불완전 state)
- `last_drain_seq` — present 시 real int (bool 제외; int() cast 가 except 안 나도록)
- `dirty_files` — present 시 list

이로써 state_is_valid 를 통과한 state 는 read_effective_mode 가 silent fallback
없이 항상 제대로 파싱 → fast-path 가 "진짜로 결정된 mode" 일 때만 skip. 어떤
필드라도 깨지면 legacy (envelope 발행).

**신규 T10** (`tests/hooks/test-state-fast-path-skip.sh`):
- last_drain_seq:"bad" (codex Round 4 repro) → 두 envelope hook legacy emit
- dirty_files 비-배열 → legacy emit
- mode 필드 누락 → legacy emit
