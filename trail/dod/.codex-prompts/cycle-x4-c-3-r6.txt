## Round 5 HIGH (class fix 완성)

Round 5 finding: `state_is_valid` 가 mode 를 "non-empty str" 로만 검사 →
`{"...","mode":"answer bogus",...}` 통과 → read_effective_mode 의
`read mode last_drain_seq` word-split 가 첫 단어 "answer" 만 취해 잘못 skip.

codex 처방대로 **mode 를 정확한 enum 으로 검증**: `answer`, `explore`,
`source_edit`, `commit` (design memo §3 + state-machine.sh 의 mode 할당 리터럴이
정확히 이 4개). 비-enum (공백 포함, 오타, None, 비-str) → state_is_valid 가
non-zero → legacy. whitespace 모호성도 함께 종결.

**Fix (state-machine.sh `state_is_valid`)**:
```
valid_modes = ("answer", "explore", "source_edit", "commit")
mode = data.get("mode")
ok = (mode in valid_modes
      and isinstance(last_drain_seq, int) and not isinstance(.., bool)
      and isinstance(dirty_files, list))
```

**T10 확장**: `mode:"answer bogus"` → 두 envelope hook 모두 legacy emit 검증 추가
(codex Round 5 가 지적한 미커버 경로).

## "answer" fallback 부류 종결 논증 (Round 1~5 누적)

read_effective_mode (fast-path 가 신뢰하는 유일한 함수, 호출처는 3개 hook 모두
`state_is_valid &&` 게이트 뒤) 가 "answer" 를 잘못 반환하던 모든 경로:

| 경로 | 종결 |
|---|---|
| state 파일 부재 (R1) | state_is_valid: `[ -f "$STATE_FILE" ]` |
| malformed JSON / unknown schema (R2) | state_is_valid: dict + schema_version==1 |
| lock 획득 실패 → "answer"+return 2 (R3) | hook capture 가 exit status 신뢰 (`&& _m=$(read_effective_mode) &&`) |
| python 부재 → "answer"+return 0 (lib) | state_is_valid: `_state_machine_python` 실패 시 return 1 |
| schema-v1 + 필드 except (last_drain_seq:"bad") (R4) | state_is_valid: last_drain_seq int / dirty_files list |
| schema-v1 + 비-enum mode word-split (R5) | state_is_valid: mode ∈ 4-enum |

state_is_valid 통과 state = {dict + schema 1 + mode ∈ enum + last_drain_seq int +
dirty_files list}. 이 경우 read_effective_mode 는 (lock OK 시) 항상 깨끗한 단일
단어 enum mode 를 return 0 으로 반환 → fast-path 는 genuine mode 일 때만 skip.
read_effective_mode 의 word-split 자체는 호출 전 enum 검증으로 비-enum mode 가
도달 불가하므로 unreachable (read_effective_mode 의 유일한 caller 3개 모두 게이트됨).
