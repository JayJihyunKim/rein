# persona

페르소나 레이어를 사용자가 직접 고르거나(선택 흐름), 질문-답변으로 새 커스텀 프리셋을 만들도록(생성 흐름) 돕는다. 이 스킬이 흐름을 완결한다 — 사용자에게 설정 파일을 직접 편집하라고 안내하지 않는다.

spec: `docs/specs/2026-07-22-persona-user-selection.md` §5 (질문 세트·순서 고정).

## 선택 흐름

사용자가 "페르소나 골라줘 / 바꿔줘 / 꺼줘" 류를 요청하면:

1. **후보 수집**:
   - 내장 프리셋 2종 (요약 1줄과 함께 제시):
     - `boss-ace` — 보스 호칭 + 과잉충성 조직 에이스. 말투만 캐릭터, 판단은 냉정.
     - `jennie` — 애교 섞인 밝은 톤, 차단·경고 순간엔 애교를 걷어내고 단칼.
   - 커스텀 프리셋: `.rein/policy/persona/*.md` 에 실존하는 파일 목록. 각 파일의 frontmatter `summary:` 를 읽어 1줄 요약으로 표시.
   - 마지막 옵션으로 **끄기(중립)** 를 항상 포함.
2. **AskUserQuestion** 으로 위 후보를 한 번에 제시하고 선택을 받는다.
3. **기록** — `mkdir -p .rein/policy` 후 `.rein/policy/persona.yaml` 에 기록:
   - 프리셋 선택 시:
     ```yaml
     enabled: true
     preset: <선택한 이름>
     ```
   - 끄기(중립) 선택 시:
     ```yaml
     enabled: false
     ```
4. **인사말 prepend + 결과 보고** — persona.yaml 에 쓰려는 상태를 기준으로(stateless — 덮어쓰기 전 이전 preset 을 읽지 않는다) 시그니처 인사말 한 줄을 상태 보고 **앞에** 붙인다:
   - **`enabled: true, preset: P`** (활성 / 전환 / **동일 재선택** 모두) → 로더로 시그니처를 얻어 상태 보고 앞에 prepend 한다. 동일 재선택도 다시 인사한다 (고정 시그니처 각인 재확인). 호출은 기존 lint 호출 관례를 따른다:
     ```bash
     GREET="$(python3 "${CLAUDE_PLUGIN_ROOT:-plugins/rein-core}/scripts/rein-policy-loader.py" --persona-greeting "<name>")"
     ```
   - **`enabled: false`** (중립 / off 전환) → 캐릭터 인사말이 아닌 **고정 평문 중립 한 줄**(이 스킬에 하드코딩, tone-neutral, 예: "중립 톤으로 돌아갈게요.")을 상태 보고 앞에 붙인다. 로더를 호출하지 않는다.
   - **fallback degrade 체인 (fail-open)** — 어떤 경로도 흐름을 중단하지 않는다: (1) 저장된 `greeting:` 을 로더로 얻는다 → (2) 출력이 비어 있으면(구 커스텀 프리셋) **후보 수집 단계에서 이미 확보한** 선택 프리셋의 summary 텍스트를 seed 로 ≤60자 시그니처 한 줄을 즉석 생성한다 → (3) summary 도 없으면 상태 보고만 낸다. 즉석 생성의 seed 는 **이미 확보한 summary 문자열뿐**이다 — 인사말을 위해 프리셋 `.md` 를 새로 열지 않는다(§경계 인사말 신뢰 경계와 정합). 내장 `boss-ace`·`jennie` 는 항상 curated greeting 이 있어 fallback 에 진입하지 않는다. 즉석 문구 불변식: tone-only + L4-clean + ≤60자.
   - **대화 언어 적응** — 저장된 `greeting:` 은 **정본(canonical)** 이다 (내장 프리셋은 한국어). 응답 언어가 정본과 다르면(예: 영어 대화) 저장 문자열을 그대로 내지 않고 **그 언어로 등가의 시그니처 인사말**을 낸다 — 캐릭터 목소리(본문 언어분기 준용)·≤60자·tone-only·L4-clean 유지. 응답 언어가 정본과 같으면 그대로 낸다. 중립 평문 한 줄·fallback 즉석 생성도 동일하게 응답 언어로 낸다. 로더는 정본만 반환하며, 언어 적응은 스킬 출력 시점의 책임이다 (저장 스키마·로더 불변).
   - **출력 예산 (간결 cap)**: 최종 출력 = `<시그니처 인사말 1줄 (≤60자)>` + 개행 + `<전환 확인 1줄>` = 짧은 2줄. 인사말은 ≤60자이며, **장황한 여러 문장 나열·이모지 스팸을 금지한다** — 느낌표·마침표·물음표로 나뉜 **짧은 다절 인사말은 허용**한다(한 호흡에 읽히는 한 줄이면 된다. 내장 boss-ace·jennie 도 다절이다). **전환 보고 어디에도** (인사말이든 상태 행이든) 인사말이 '무엇인지' 설명하지 않는다 — "이건 전환 신호다 / 1회성이다 / 다음 세션부터 적용" 류의 메타 서술을 붙이지 않는다. 전환 확인은 "<이름>로 바꿨어요" 수준의 한 줄로 끝낸다.

## 생성 흐름 (7문항 Q&A — 순서 고정)

사용자가 "페르소나 만들어줘" 류를 요청하면, AskUserQuestion(자유 입력 항목은 채팅 답변)으로 아래 7문항을 **순서 그대로** 진행한다:

1. **이름** — 파일명이 된다. 영문 소문자·숫자·하이픈, 32자 이내. **내장 프리셋(`boss-ace`, `jennie`)과 동일 이름은 사용할 수 없음을 질문 시점에 사전 고지**한다.
2. **호칭** — 사용자를 뭐라고 부를까 (자유 입력).
3. **캐릭터 컨셉 한 줄** — 예: "츤데레 선배", "무뚝뚝한 집사".
4. **표현 수위** — `절제` / `보통` / `진하게` 3단계 중 선택.
5. **언어분기** — EN 응답 시 톤. 미지정 시 기본: 직역 금지 + self-aware 완화 (내장 프리셋 관례).
6. **차단·경고 순간의 말투** — 판단의 냉정·직설은 불변층이 고정하므로, 캐릭터가 그 순간을 "어떤 말투로" 전달하는지만 묻는다.
7. **예시 멘트** — 답변을 반영한 참고 예시 2개(Done 1 + 차단 1)를 생성기가 초안으로 제시하고, 사용자가 수정·삭제할 수 있다.

### 초안 제시 → 확인 → lint → 저장

1. 7문항 답변으로 본문 초안을 작성해 **전문을 채팅에 제시**한다. 구조는 내장 프리셋 준용:
   - frontmatter `summary:` 는 캐릭터 컨셉 기반으로 **자동 작성**한다 (사용자에게 묻지 않음).
   - frontmatter `greeting:` 은 캐릭터 컨셉(Q3)에서 파생한 시그니처 인사말 한 줄로 **자동 작성**해 전문 초안에 포함하고, 참고 예시 멘트(Q7)와 나란히 보이게 제시한다. 사용자가 수정·삭제할 수 있으며, 그 후 lint → 저장으로 진행한다. **summary 자동 작성 관례와 동일하게 greeting 도 자동 작성하며, 8번째 질문을 만들지 않는다 — 7문항의 순서·개수는 불변이다.**
   - 본문: 캐릭터 / 언어분기 / 간결 cap / 참고 예시.
   - **불변 조항(판단 냉정·응답 규칙 우선 등)은 본문에 넣지 않는다** — 불변층이 구조로 담당하므로 중복 금지.
   - 생성기가 산출한 `greeting:` 은 저장 파일의 일부이므로 lint L3/L4 대상이다 (정확한 `---`(LF) fence · tone-only · L4-clean). 사용자 편집이 L4 를 위반하거나 (P)∧¬(A) fence 를 넣으면 강화 lint 가 저장 전에 잡고, lint 를 우회해 저장하더라도 로더의 런타임 경계가 로드를 거부한다.
2. 사용자 확인을 받는다 (수정 요청 시 해당 부분만 고쳐 재제시).
3. 초안을 임시 파일에 두고 lint 를 실행한다:
   ```bash
   LINT="${CLAUDE_PLUGIN_ROOT:-plugins/rein-core}/scripts/rein-persona-lint.py"
   python3 "$LINT" --name "<name>" --body-file "<draft-tmp>"
   ```
4. **lint 통과 시에만 저장**: `mkdir -p .rein/policy/persona` 후 `.rein/policy/persona/<name>.md` 로 저장. 통과 전에는 어떤 부분 파일도 저장하지 않는다.
5. **lint 실패 시**: 위반 규칙(L1~L5)과 원인이 된 답변 항목을 짚어 **해당 항목만 재질문**하고 초안을 재생성한다. 전체 재시작 금지.
6. **동명 커스텀 존재 시**: `.rein/policy/persona/<name>.md` 가 이미 있으면 덮어쓰기 여부를 AskUserQuestion 으로 확인한 뒤 진행한다.

### 적용 질문

저장 후 "지금 적용할까요?" 를 묻는다:
- **예** → 선택 흐름과 동일하게 `.rein/policy/persona.yaml` 에 `enabled: true` + `preset: <name>` 기록.
- **아니오** → 저장만 하고 종료 (persona.yaml 은 건드리지 않음).

## 경계

- **lint 우회 저장 금지** — lint 실패 상태의 본문을 어떤 경로로도 `.rein/policy/persona/` 에 쓰지 않는다.
- 이 스킬이 수정하는 설정 파일은 `.rein/policy/persona.yaml` 과 `.rein/policy/persona/<name>.md` 뿐이다. 그 외 파일 수정 금지.
- **인사말 신뢰 경계 (정적 계약)**: 활성 프리셋 인사말은 오직 로더 `--persona-greeting <name>` 로 얻고, fallback seed 는 후보 수집 단계에서 **이미 확보한** summary 문자열로만 쓴다. 인사말 기능을 위해 프리셋 `.md` 파일을 **새로 열지 않는다**. (후보 수집이 각 커스텀 `summary:` 를 읽는 것은 이 기능이 건드리지 않는 기존 동작이다 — 제거·변경 아님.)
- **불변층 종속**: 인사말은 불변층(`rules/persona/_invariant.md`) 아래 tone-only 로 종속된다 — 판단·경고·차단(게이트 hard warning 포함)을 약화·대체·완곡화하지 않으며, 충돌 시 응답 규칙이 항상 이긴다.
- 내부 운영 경로(trail/, hooks/ 등)를 사용자 채팅 본문에 노출하지 않는다 — 결과는 평문으로 보고.
