# Skill: incidents-to-agent

## 새 에이전트 감지 기준 (3가지 모두 충족 시만)
1. 동일 작업 유형 기존 에이전트 self-review 실패 **3회 이상**
2. 작업 유형이 **한 문장으로 설명 가능**하고 기존 에이전트와 명확히 구분
3. 기존 에이전트 + 하위 AGENTS.md 조합으로 해결 **불가능**

## 입력
- `trail/incidents/` 전체 스캔

## 실행 절차

### Step 1: 패턴 분류
```
[ ] 작업 유형별 그룹화
[ ] 각 그룹 반복 횟수 카운트
[ ] 3회 이상 반복 그룹 식별
```

### Step 2: 에이전트 필요성 판단
```
[ ] 기존 에이전트로 해결 가능한가?
[ ] 하위 AGENTS.md 규칙 추가로 해결 가능한가?
[ ] 역할을 한 문장으로 설명 가능한가?
[ ] 기존 에이전트와 명확히 구분되는가?
```

### Step 3: 후보 생성 (기준 충족 시만)
`trail/agent-candidates/{name}.md`:
```markdown
# 에이전트 후보: {name}
## 역할 (한 문장): [역할]
## 근거: INC-NNN, INC-MMM, INC-LLL (N회)
## 기존 에이전트로 해결 불가 이유: [이유]
## DoD: [완료 기준]
```

## 주의
- 에이전트 수 증가 = 조정 비용 + 토큰 비용 + 충돌 위험 증가
- 불확실하면 에이전트 생성보다 규칙 추가 선택
- 사람의 최종 승인 없이 에이전트를 활성화하지 않는다
