# Workflow: build-from-scratch
> 새 서비스, 모듈, 앱을 처음부터 만들 때 사용

## 실행 절차

### Step 1: 요구사항 명확화
```
[ ] 서비스 목적 한 문장 정의
[ ] 주요 기능 목록 (MVP 범위)
[ ] 기술 스택 결정 → trail/decisions/DEC-NNN.md에 기록
[ ] 외부 의존성 목록
```

### Step 2: DoD 작성
```markdown
## DoD: [서비스명] 초기 구조
- [ ] 디렉토리 구조 생성 완료
- [ ] 진입점 동작 확인
- [ ] 기본 에러 처리 구조 포함
- [ ] 환경변수 관리 구조 포함 (.env.example)
- [ ] 하위 AGENTS.md 작성
- [ ] 기본 테스트 구조 포함
```

### Step 3: 디렉토리 구조 설계
```
[ ] 폴더 구조 스케치 (10줄 이내)
[ ] 각 폴더 역할 한 줄 설명
[ ] 의존 방향 정의 (단방향 원칙)
```

### Step 4: 구현 (incremental)
```
[ ] 진입점만 먼저 → 실행 확인
[ ] 핵심 기능 추가 → 동작 확인
[ ] 에러 처리 추가
[ ] 테스트 작성
[ ] 문서화
```

### Step 5: 완료 검증
```
[ ] DoD 전체 충족
[ ] 하위 AGENTS.md 작성 완료
[ ] trail/decisions/DEC-NNN.md 기록
[ ] trail/index.md 갱신
```

## 기술 결정 기록 형식 (DEC-NNN.md)
```markdown
# DEC-NNN: [결정 제목]
- 날짜: YYYY-MM-DD
- 결정: [무엇을 선택]
- 이유: [왜 이 선택]
- 대안: [고려한 다른 옵션]
- 영향: [이 결정의 영향]
```
