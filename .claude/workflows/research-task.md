# Workflow: research-task
> 기술 조사, 라이브러리 비교, 아키텍처 결정을 위한 조사 절차

## 실행 절차

### Step 1: 조사 범위 정의
```
[ ] 조사 목적 한 문장으로 명시
[ ] 조사 결과로 내릴 결정 명시
[ ] 조사 기간/깊이 한정 (시간 무한정 소비 방지)
```

### Step 2: DoD 작성
```markdown
## DoD: [조사 주제]
- [ ] 비교 대상 항목 전체 검토
- [ ] 각 옵션 장단점 정리
- [ ] 권장 결론 및 이유 명시
- [ ] SOT/decisions/DEC-NNN.md에 기록
```

### Step 3: 조사 수행
```
[ ] 공식 문서 우선 확인
[ ] 실사용 사례 (GitHub, 기술 블로그) 확인
[ ] 최신 버전/업데이트 상태 확인
[ ] 라이선스/비용 확인
```

### Step 4: 결과 정리
```markdown
| 항목 | 옵션 A | 옵션 B | 옵션 C |
|------|--------|--------|--------|
| 성능 | | | |
| 학습곡선 | | | |
| 커뮤니티 | | | |
| 라이선스 | | | |
| 결론 | ✅ 권장 | | |
```

### Step 5: 결정 기록
```
[ ] SOT/decisions/DEC-NNN.md 작성
[ ] 필요시 AGENTS.md 또는 하위 AGENTS.md에 반영
[ ] SOT/index.md 갱신
```
